4. Combobox-specific shadow DOM patterns

These scenarios test the interaction between aria-activedescendant, aria-controls, and owned listbox elements when they live inside a shadow root. This is where combobox behavior diverges most from a simple text input.

4a. aria-activedescendant with shadow-scoped option IDs

The shadow input uses aria-activedescendant pointing at option IDs within the same shadow root. Verify the a11y tree shows the active option.

The input must declare ownership of the listbox via aria-controls. Both live inside the shadow root. Verify the relationship is exposed in the a11y tree.
