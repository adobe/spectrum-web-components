# Button consumer migration guide

Replace `<sp-button>` with `<swc-button>`, update the import, and migrate renamed attributes.

## What changed

### Renamed

| Area                        | Spectrum 1 (`sp-button`)                       | Spectrum 2 (`swc-button`)                    |
| --------------------------- | ---------------------------------------------- | -------------------------------------------- |
| Tag                         | `sp-button`                                    | `swc-button`                                 |
| Import path                 | `@spectrum-web-components/button/sp-button.js` | `@adobe/spectrum-wc/button`                  |
| Visual treatment            | `treatment="fill"` / `treatment="outline"`     | `fill-style="fill"` / `fill-style="outline"` |
| Accessible name (icon-only) | `label="..."`                                  | `accessible-label="..."`                     |
| Text truncation             | `no-wrap`                                      | `truncate`                                   |
| CSS custom props            | `--mod-button-*` / `--spectrum-button-*`       | `--swc-button-*` (see [Styling](#styling))   |

### Added in Spectrum 2

| Addition                      | Notes                                                                                                                                              |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `justified` boolean attribute | Stretches the button to fill its container's inline size. The container must allow the element to grow.                                            |
| `pending-label` derivation    | When `pending` is set and no `pending-label` is provided, the accessible name is automatically derived as `"[label], busy"` (e.g. `"Save, busy"`). |

### Removed in Spectrum 2

| Removed                                                    | Replacement                                                              |
| ---------------------------------------------------------- | ------------------------------------------------------------------------ |
| `href`, `target`, `download`, `referrerpolicy`, `rel`      | Use a native `<a>` element with global button styles (see step 5 below). |
| `quiet` boolean attribute                                  | Use `fill-style="outline"` instead.                                      |
| Variant aliases: `cta`, `overBackground`, `white`, `black` | Use `variant` + `static-color` (see step 3 below).                       |
| `label` attribute                                          | Use `accessible-label` instead.                                          |
| `--mod-button-*` custom properties                         | Use `--swc-button-*` equivalents (see [Styling](#styling)).              |

## Update your code

### 1. Update the import

```js
// Before
import '@spectrum-web-components/button/sp-button.js';
// After
import '@adobe/spectrum-wc/button';
```

### 2. Rename the tag and `treatment` attribute

```html
<!-- Before -->
<sp-button variant="primary" treatment="fill">Save</sp-button>
<sp-button variant="primary" treatment="outline">Cancel</sp-button>
<sp-button variant="primary" quiet>Cancel</sp-button>
<!-- After -->
<swc-button variant="primary" fill-style="fill">Save</swc-button>
<swc-button variant="primary" fill-style="outline">Cancel</swc-button>
<swc-button variant="primary" fill-style="outline">Cancel</swc-button>
```

`fill-style="fill"` is the default and may be omitted.

### 3. Replace deprecated variant aliases

The deprecated aliases `cta`, `overBackground`, `white`, and `black` are removed. Migrate to canonical `variant` + `static-color` combinations:

```html
<!-- Before -->
<sp-button variant="cta">Save</sp-button>
<sp-button variant="overBackground">Action</sp-button>
<sp-button variant="white">Action</sp-button>
<sp-button variant="black">Action</sp-button>
<!-- After -->
<swc-button variant="accent">Save</swc-button>
<swc-button variant="primary" static-color="white">Action</swc-button>
<swc-button variant="primary" static-color="white">Action</swc-button>
<swc-button variant="primary" static-color="black">Action</swc-button>
```

`static-color` is only supported with `primary` and `secondary` variants.

### 4. Rename `label` to `accessible-label`

```html
<!-- Before -->
<sp-button label="Add item">
  <svg slot="icon">...</svg>
</sp-button>
<!-- After -->
<swc-button accessible-label="Add item">
  <svg slot="icon">...</svg>
</swc-button>
```

### 5. Migrate link-mode usage to native anchors

`<sp-button href="...">` is removed. Use a native `<a>` element with [global button styles](/docs/guides-customization-global-element-styling--readme) instead:

```html
<!-- Before -->
<sp-button href="/dashboard" variant="primary">Go to dashboard</sp-button>
<!-- After -->
<a href="/dashboard" class="swc-Button">Go to dashboard</a>
```

Import the global button stylesheet:

```js
import '@adobe/spectrum-wc/global-button.css';
```

### 6. Rename `no-wrap` to `truncate`

```html
<!-- Before -->
<sp-button no-wrap>Long label that should not wrap</sp-button>
<!-- After -->
<swc-button truncate>Long label that should not wrap</swc-button>
```

## Accessibility

- **Icon-only buttons** must have `accessible-label` on the host. See [step 4](#4-rename-label-to-accessible-label).
- **Pending state**: when `pending` is set, the button announces as `"[label], busy"` by default. Supply `pending-label` to override with a more specific description of the in-progress operation.
- **Disabled vs. pending**: use `pending` to keep the button focusable while an operation is in progress. Use `disabled` to remove it from the tab order entirely. Do not set both at the same time.
- **Navigation**: do not use `<swc-button>` for navigation. Buttons activate on both Enter and Space; links activate on Enter only. Use a native `<a>` with global button styles instead (see [step 5](#5-migrate-link-mode-usage-to-native-anchors)).
- **Form submission**: `<swc-button>` behaves as `type="button"` only. For `submit` and `reset`, use a native `<button>` with [global button styles](/docs/guides-customization-global-element-styling--readme) until form-associated behavior is added in a future release.
