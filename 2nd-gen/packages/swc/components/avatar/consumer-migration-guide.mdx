# Avatar consumer migration guide

Replace `<sp-avatar>` with `<swc-avatar>`, update the import, and rename several attributes. The `href` (link) capability has been removed.

## Installation

```bash
# Remove
yarn remove @spectrum-web-components/avatar

# Add
yarn add @adobe/spectrum-wc
```

Update your import:

```js
// Before
import '@spectrum-web-components/avatar/sp-avatar.js';

// After
import '@adobe/spectrum-wc/avatar';
```

> `@adobe/spectrum-wc` is a monolithic package. Importing via subpath (e.g. `@adobe/spectrum-wc/avatar`) loads only that component's bundle.

### Renamed

| Area                      | Spectrum 1 (`sp-avatar`)                       | Spectrum 2 (`swc-avatar`)                                                     |
| ------------------------- | ---------------------------------------------- | ----------------------------------------------------------------------------- |
| Tag                       | `sp-avatar`                                    | `swc-avatar`                                                                  |
| Import path               | `@spectrum-web-components/avatar/sp-avatar.js` | `@adobe/spectrum-wc/avatar`                                                   |
| `label` attribute         | Accessible label for the image                 | Renamed to `alt` â€” see [step 2](#2-rename-label-to-alt)                       |

| CSS custom props          | `--mod-avatar-*`                               | `--swc-avatar-*` (see [Styling](#styling))                                    |

| Addition                     | Notes                                                |
| ---------------------------- | ---------------------------------------------------- |
| `outline` boolean attribute  | Renders a visible border around the avatar           |
| `disabled` boolean attribute | Applies reduced opacity to indicate a disabled state |
| Sizes 800â€“1500               | Valid `size` values expanded from 50â€“700 to 50â€“1500  |

## Update your code

### 1. Rename the tag

```html
<!-- Before -->
<sp-avatar src="photo.jpg" label="Jane Smith"></sp-avatar>
<!-- After -->
<swc-avatar src="photo.jpg" alt="Jane Smith"></swc-avatar>
```

### 2. Rename `label` to `alt`

The `label` attribute has been renamed to `alt` to align with the native `<img>` convention.

```html
<!-- Before -->
<sp-avatar src="photo.jpg" label="Jane Smith"></sp-avatar>
<!-- After -->
<swc-avatar src="photo.jpg" alt="Jane Smith"></swc-avatar>
```

### 3. Rename `is-decorative` to `decorative`

The `is-decorative` attribute has been renamed to `decorative`. Use `alt=""` when you want an accessible but unnamed image (equivalent to `<img alt="">`); use `decorative` when you want to hide the element from assistive technology entirely.

```html
<!-- Before -->
<sp-avatar src="photo.jpg" is-decorative></sp-avatar>
<!-- After -->
<swc-avatar src="photo.jpg" decorative></swc-avatar>
```

### 4. Replace `href` with an anchor wrapper

The built-in link capability has been removed. Wrap `<swc-avatar>` in a standard `<a>` element to achieve the same result.

```html
<!-- Before -->
<sp-avatar src="photo.jpg" label="Jane Smith" href="/profile/jane"></sp-avatar>
<!-- After -->
<a href="/profile/jane" aria-label="Jane Smith's profile">
  <swc-avatar src="photo.jpg" alt="Jane Smith"></swc-avatar>
</a>
```

### 5. Verify your `size` value

The default `size` changed from `100` to `500`. If you relied on the default, set the size explicitly.

```html
<!-- Before (implicitly size="100") -->
<sp-avatar src="photo.jpg" label="Jane Smith"></sp-avatar>
<!-- After â€” set size explicitly to preserve the previous default -->
<swc-avatar src="photo.jpg" alt="Jane Smith" size="100"></swc-avatar>
```

## Accessibility

- Always provide a descriptive `alt` attribute that identifies the person or entity the avatar depicts (e.g., `alt="Jane Smith"`).
- Set `alt=""` to mark the avatar as decorative when the surrounding context already conveys the same information â€” the component will then be hidden from assistive technology.
- A developer warning is logged if neither `alt` nor `decorative` is set, signaling that the accessible name is missing.
- If you replace `href` with an `<a>` wrapper, add `aria-label` to the anchor when the link destination is not obvious from surrounding content.

### Removed `--mod-avatar-*` properties

Replace every `--mod-avatar-*` property with its `--swc-avatar-*` equivalent:

| Removed (Spectrum 1)        | Replacement (Spectrum 2)     |
| --------------------------- | ---------------------------- |
| `--mod-avatar-size`         | `--swc-avatar-size`          |
| `--mod-avatar-border-color` | `--swc-avatar-outline-color` |
| `--mod-avatar-border-width` | `--swc-avatar-outline-width` |
