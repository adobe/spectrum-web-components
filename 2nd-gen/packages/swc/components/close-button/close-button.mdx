## Anatomy

A close button consists of:

1. **Button** ‚Äî native `<button type="button">` with delegated focus; handles pointer and keyboard activation
2. **Icon** ‚Äî cross icon (`aria-hidden="true"`); size follows the `size` attribute

#### Properties

- **`accessible-label`** ‚Äî required accessible name forwarded to the inner button as `aria-label`
- **`size`** ‚Äî visual size of the hit target and icon scale (`s`, `m`, `l`, `xl`)
- **`static-color`** ‚Äî static color treatment for display over colored or image backgrounds (`white`, `black`)

## Options

### Sizes

Close buttons come in four sizes to fit various contexts:

- **Small (`s`)** ‚Äî compact chrome such as dense toolbars or inline banners
- **Medium (`m`)** ‚Äî default size for dialogs, toasts, and most dismiss affordances
- **Large (`l`)** ‚Äî larger hit target when space allows
- **Extra large (`xl`)** ‚Äî maximum visibility for prominent dismiss controls

### Static colors

Use the `static-color` attribute when displaying over images or colored backgrounds:

- **white** ‚Äî use on dark or colored backgrounds for better contrast
- **black** ‚Äî use on light backgrounds for better contrast

Close buttons support enabled and disabled states. Hover, active, and keyboard focus styles apply only when the control is not disabled.

## Accessibility

### Features

The `<swc-close-button>` element implements several accessibility features:

#### Keyboard navigation

- **Tab** ‚Äî moves focus to the inner dismiss button via delegated focus
- **Enter** or **Space** ‚Äî activates the dismiss action
- **Focus indicator** ‚Äî visible `:focus-visible` ring on keyboard focus

#### ARIA implementation

1. **Native button** ‚Äî renders a real `<button type="button">` inside the shadow root
2. **Accessible name** ‚Äî provided via `accessible-label` and forwarded as `aria-label` on the inner button
3. **Decorative icon** ‚Äî cross icon is marked `aria-hidden="true"` so screen readers announce the button name only
4. **Disabled state** ‚Äî `disabled` removes activation and applies disabled visuals on the inner button

#### Visual accessibility

- Hit target size scales with the `size` attribute
- Static color variants maintain contrast on colored backgrounds
- Transparent border and outline focus treatment support high contrast mode without custom forced-color overrides

### Best practices

- Always provide a descriptive accessible name such as "Close" or "Close dialog" via `accessible-label` ‚Äî not a clear/reset verb
- Use `static-color="white"` or `static-color="black"` when the default neutral icon lacks contrast on the background
- Test keyboard focus order when placing close buttons in dialog headers or banner corners
