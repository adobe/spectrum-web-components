# Asset consumer migration guide

Replace `<sp-asset>` with `<swc-asset>` and update the import. The public API is unchanged.

## Installation

```bash
# Remove
yarn remove @spectrum-web-components/asset

# Add
yarn add @adobe/spectrum-wc
```

Update your import:

```js
// Before
import '@spectrum-web-components/asset/sp-asset.js';

// After
import '@adobe/spectrum-wc/asset';
```

> `@adobe/spectrum-wc` is a monolithic package. Importing via subpath (e.g. `@adobe/spectrum-wc/asset`) loads only that component's bundle.

## What changed

### Renamed

| Area        | Spectrum 1 (`sp-asset`)                      | Spectrum 2 (`swc-asset`)   |
| ----------- | -------------------------------------------- | -------------------------- |
| Tag         | `sp-asset`                                   | `swc-asset`                |
| Import path | `@spectrum-web-components/asset/sp-asset.js` | `@adobe/spectrum-wc/asset` |

### Removed in Spectrum 2

| Removed                                                                                                                                                                                             | Replacement                                                                          |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| `--mod-asset-folder-background-color`, `--mod-asset-file-background-color`, `--mod-asset-icon-outline-color`, `--mod-asset-icon-min-width`, `--mod-asset-icon-max-width`, `--mod-asset-icon-margin` | No public CSS custom properties in Spectrum 2 — remove all `--mod-asset-*` overrides |

## Update your code

### 1. Rename the tag

```html
<!-- Before -->
<sp-asset variant="file" label="README.md"></sp-asset>
<!-- After -->
<swc-asset variant="file" label="README.md"></swc-asset>
```

The `variant` attribute (`'file'` | `'folder'`), `label` attribute, and the default slot are all unchanged:

```html
<!-- Before -->
<sp-asset variant="folder" label="My project"></sp-asset>
<sp-asset label="Custom content">
  <img src="preview.png" alt="Preview" />
</sp-asset>

<!-- After -->
<swc-asset variant="folder" label="My project"></swc-asset>
<swc-asset label="Custom content">
  <img src="preview.png" alt="Preview" />
</swc-asset>
```

### 2. Remove `--mod-asset-*` overrides

Spectrum 2 does not expose public CSS custom properties on `<swc-asset>`. Any `--mod-asset-*` overrides in your stylesheets must be removed.

```css
/* Before */
sp-asset {
  --mod-asset-folder-background-color: hotpink;
  --mod-asset-icon-min-width: 60px;
}

/* After — remove these declarations entirely */
```

## Accessibility

- When `variant="file"` or `variant="folder"`, always provide a descriptive `label` attribute. It is used as the accessible name for the icon so screen readers can announce what the asset represents.
- When using the default slot for custom content, the slotted content is responsible for its own accessible labeling (e.g., an `<img>` with a descriptive `alt` attribute).
