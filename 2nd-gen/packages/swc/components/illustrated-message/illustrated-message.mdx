## Anatomy

An illustrated message consists of:

- **Illustration**, provided through the default slot: decorative or informative SVG displayed above (vertical) or beside (horizontal) the heading and description. Decorative SVGs must have `aria-hidden="true"`. Informative SVGs must have `role="img"` and `aria-label`.
- **Heading**, provided through the `heading` slot: a consumer-provided `<h2>`–`<h6>` element. The component does not own the heading tag; the consumer chooses the level to match the document outline. The component warns in development if a non-heading element is used.
- **Description** (optional), provided through the `description` slot: supporting text that elaborates on the heading. May contain phrasing content or links.
- **Actions** (optional), provided through the `actions` slot: one or more interactive controls displayed below the description. The component automatically propagates its `size` to slotted elements.

## Options

### Sizes

Illustrated messages come in three sizes to fit various contexts:

Each size uses a progressively larger heading/description type scale from small to medium to large.

- **Small (`s`)**: 96px illustration, smaller heading/description type, for space-constrained contexts
- **Medium (`m`)**: 96px illustration, default type scale
- **Large (`l`)**: 160px illustration, larger heading/description type, a tighter illustration-to-content gap, for prominent empty states

### Orientation

Illustrated messages support two layout orientations:

- **Vertical** (default): illustration stacked above the heading and description, centered. Use for full-page or centered empty states.
- **Horizontal**: illustration beside the heading and description in a row, left-aligned. Use for inline or sidebar empty states.

<Canvas of={IllustratedMessageStories.Orientation} />

### Heading levels

The `heading` slot accepts any `<h2>`–`<h6>` element. Choose the level that fits the surrounding document outline. The component validates but does not restrict the heading level.

Common patterns:

- `<h2>` for page-level empty states with no other headings in scope
- `<h3>` for empty states nested inside a page section with its own `<h2>`
- `<h4>` or deeper for empty states inside nested panels or cards

## Behaviors

### Actions

The `actions` slot accepts one or more interactive controls displayed below the description. The illustrated message automatically propagates its `size` to every slotted element, so slotted buttons match the message size automatically. Use a single `<swc-button>` for a primary call-to-action, or a `<swc-button-group>` when two actions are needed.

## Accessibility

### Features

The `<swc-illustrated-message>` element implements the following accessibility features:

1. **Heading ownership**: the consumer provides the `<h2>`–`<h6>` element directly in the `heading` slot, preserving full control over heading level and ensuring it participates in the document outline.
2. **Illustration intent**: SVGs in the default slot must declare their accessibility intent explicitly via `aria-hidden` or `role="img"`.
3. **No redundant ARIA**: the component adds no `role` or `aria-*` attributes of its own; semantics come entirely from slotted content.

### Best practices

- Decorative illustrations (most common): add `aria-hidden="true"` so screen readers skip the graphic and move directly to the heading.
- Informative illustrations: add `role="img"` and `aria-label` (or an inline `<title>`) so screen readers announce the illustration's meaning before reading the heading and description.
- Choose the heading level (`h2`–`h6`) that fits the surrounding document outline, not based on visual size.
