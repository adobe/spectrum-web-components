# Drop zone migration guide

Replace `<sp-dropzone>` with `<swc-dropzone>`, rename `isDragged`/`isFilled` to `dragged`/`filled`, move uploaded-state content into the new `filled-content` slot, and add a required accessible name.

## Installation

```bash
# Remove
yarn remove @spectrum-web-components/dropzone

# Add
yarn add @adobe/spectrum-wc
```

Update your import:

```js
// Before
import '@spectrum-web-components/dropzone/sp-dropzone.js';

// After
import '@adobe/spectrum-wc/components/dropzone/swc-dropzone.js';
```

> `@adobe/spectrum-wc` is a monolithic package. Importing via subpath (e.g. `@adobe/spectrum-wc/components/dropzone/swc-dropzone.js`) registers and loads only that component's bundle.

## What changed

### Renamed

| Area             | Spectrum 1 (`sp-dropzone`)                                                                       | Spectrum 2 (`swc-dropzone`)                                                                          |
| ---------------- | ------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| Tag              | `sp-dropzone`                                                                                    | `swc-dropzone`                                                                                       |
| Import path      | `@spectrum-web-components/dropzone`                                                              | `@adobe/spectrum-wc/components/dropzone/swc-dropzone.js`                                             |
| Dragged property | `isDragged` (attribute stays `dragged`)                                                          | `dragged`                                                                                            |
| Filled property  | `isFilled` (attribute stays `filled`; setting it in JS did not reflect the attribute)            | `filled` (now reflects the attribute correctly)                                                      |
| Events           | `sp-dropzone-should-accept`, `sp-dropzone-dragover`, `sp-dropzone-dragleave`, `sp-dropzone-drop` | `swc-dropzone-should-accept`, `swc-dropzone-dragover`, `swc-dropzone-dragleave`, `swc-dropzone-drop` |
| CSS custom props | `--mod-drop-zone-*` and `--mod-illustrated-message-*` passthrough                                | `--swc-dropzone-*` (see [Styling](#styling))                                                         |

If your code only ever set `dropzone.isFilled = true` in JavaScript (never `setAttribute('filled', ...)`), the filled background tint and the `filled-content` slot swap never actually applied in Spectrum 1, because the attribute was never reflected. In Spectrum 2 the same JavaScript now works correctly, so you may see this visual change appear for the first time after upgrading with no code changes on your part.

### Added in Spectrum 2

| Addition              | Notes                                                                                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `size` attribute      | `'s' \| 'm' \| 'l'`, default `'m'`. Scales the container and a slotted `swc-illustrated-message`'s illustration.                                                   |
| `filled-content` slot | Dedicated slot for uploaded-state content, shown automatically when `filled` is `true`. See [step 4](#4-move-uploaded-state-content-into-the-filled-content-slot). |

### Removed in Spectrum 2

| Removed                                                               | Replacement                                                                                                                                                                                                                                                            |
| --------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Public methods `onDragOver`, `onDragLeave`, `onDrop`                  | Not part of the public API. Listen for `swc-dropzone-should-accept`, `swc-dropzone-dragover`, `swc-dropzone-dragleave`, and `swc-dropzone-drop` instead of overriding these methods.                                                                                   |
| `DropzoneEventDetail` type                                            | Use `DragEvent` directly for `swc-dropzone-should-accept`, `swc-dropzone-dragover`, and `swc-dropzone-drop`. `swc-dropzone-dragleave` now dispatches a `{ clientX, clientY, relatedTarget }` detail instead of a `DragEvent`; see [step 3](#3-rename-event-listeners). |
| `--mod-drop-zone-*` / `--mod-illustrated-message-*` custom properties | `--swc-dropzone-*` for the small set of exposed properties (see [Styling](#styling)). Most 1st-gen modifier properties have no replacement.                                                                                                                            |

## Update your code

### 1. Rename the tag

```html
<!-- Before -->
<sp-dropzone id="dropzone" style="width: 400px;">
  <sp-illustrated-message heading="Drag and drop your file">
    <svg>...</svg>
  </sp-illustrated-message>
  <sp-button variant="accent">Browse files</sp-button>
</sp-dropzone>

<!-- After -->
<swc-dropzone id="dropzone" aria-label="Upload files" style="width: 400px;">
  <swc-illustrated-message>
    <svg aria-hidden="true">...</svg>
    <h2 slot="heading">Drag and drop your file</h2>
    <swc-button slot="actions" variant="accent">Browse files</swc-button>
  </swc-illustrated-message>
</swc-dropzone>
```

### 2. Rename JS property references

```js
// Before
dropzone.isDragged;
dropzone.isFilled = true;

// After
dropzone.dragged;
dropzone.filled = true;
```

Lit template bindings (`?dragged=‚Ä¶`, `?filled=‚Ä¶`) and the `dragged`/`filled` HTML attributes are unchanged.

### 3. Rename event listeners

```js
// Before
dropzone.addEventListener('sp-dropzone-drop', handleDrop);

// After
dropzone.addEventListener('swc-dropzone-drop', handleDrop);
```

Rename `sp-dropzone-should-accept` and `sp-dropzone-dragover` the same way.

Rename `sp-dropzone-dragleave` to `swc-dropzone-dragleave` too, but update how you read its detail: it no longer carries the full `DragEvent`, only `{ clientX, clientY, relatedTarget }`.

```js
// Before
dropzone.addEventListener('sp-dropzone-dragleave', (event) => {
  console.log(event.detail.dataTransfer);
});

// After
dropzone.addEventListener('swc-dropzone-dragleave', (event) => {
  console.log(event.detail.clientX, event.detail.clientY);
});
```

### 4. Move uploaded-state content into the `filled-content` slot

In Spectrum 1, setting `filled` only changed styling; the same slotted content stayed in the DOM and you updated it in place (for example, changing the illustrated message's heading). In Spectrum 2, setting `filled` swaps the entire default slot out for a separate `filled-content` slot. Provide the uploaded-state content there instead of mutating the default slot's content.

```html
<!-- Before -->
<sp-dropzone id="dropzone">
  <sp-illustrated-message id="message" heading="Drag and drop your file">
    <svg>...</svg>
  </sp-illustrated-message>
  <sp-button variant="accent">Browse files</sp-button>
</sp-dropzone>
<script type="module">
  // On drop: dropzone.setAttribute('filled', true); message.heading = 'You dropped something!';
</script>

<!-- After -->
<swc-dropzone id="dropzone" aria-label="Upload files">
  <swc-illustrated-message>
    <svg aria-hidden="true">...</svg>
    <h2 slot="heading">Drag and drop your file</h2>
    <swc-button slot="actions" variant="accent">Browse files</swc-button>
  </swc-illustrated-message>
  <div slot="filled-content">
    <span>report-q4.pdf uploaded</span>
    <swc-button size="s" variant="secondary" class="replace-button">
      Replace file
    </swc-button>
  </div>
</swc-dropzone>
<script type="module">
  // On drop: dropzone.setAttribute('filled', 'true');
  // Update the filled-content element's text instead of the illustrated message.
  // Keep a control such as "Replace file" reachable: the default slot's browse
  // button is hidden once filled, so it can no longer receive focus.
</script>
```

### 5. Fix consumer-facing accessibility gaps

An accessible name is now required. Add `aria-label` or `aria-labelledby`; a development-mode warning fires if neither is present.

```html
<!-- Before -->
<sp-dropzone>...</sp-dropzone>

<!-- After -->
<swc-dropzone aria-label="Upload files">...</swc-dropzone>
```

Remove any `role` attribute you previously added. `role="group"` is now fixed on the host and is not author-overridable.

```html
<!-- Before -->
<sp-dropzone role="group" aria-label="Upload files">...</sp-dropzone>

<!-- After -->
<swc-dropzone aria-label="Upload files">...</swc-dropzone>
```

Replace the `javascript:;` link pattern with a real button or link that triggers the hidden file input from an event listener. Scope the lookup to each dropzone instance rather than hardcoding ids, so the pattern still works when a page has more than one `<swc-dropzone>`.

```html
<!-- Before -->
<sp-dropzone>
  <label for="file-input">
    <sp-link
      href="javascript:;"
      onclick="this.parentElement.nextElementSibling.click()"
    >
      Select a File
    </sp-link>
  </label>
  <input type="file" id="file-input" style="display: none" />
</sp-dropzone>

<!-- After -->
<swc-dropzone aria-label="Upload files">
  <swc-illustrated-message>
    <svg aria-hidden="true">...</svg>
    <h2 slot="heading">Drag and drop your file</h2>
    <swc-button class="browse-button" slot="actions" variant="accent">
      Browse files
    </swc-button>
  </swc-illustrated-message>
  <input
    type="file"
    class="file-input"
    aria-label="Choose a file"
    style="display: none"
  />
</swc-dropzone>
<script type="module">
  document.querySelectorAll('swc-dropzone').forEach((dropzone) => {
    const fileInput = dropzone.querySelector('.file-input');
    dropzone
      .querySelector('.browse-button')
      .addEventListener('click', () => fileInput.click());
  });
</script>
```

### 6. (Optional) Adopt the `size` attribute

```html
<!-- Before -->
<swc-dropzone aria-label="Upload files">...</swc-dropzone>

<!-- After -->
<swc-dropzone size="l" aria-label="Upload files">...</swc-dropzone>
```

## Accessibility

- **Accessible name is required.** Add `aria-label` or `aria-labelledby`. See [step 5](#5-fix-consumer-facing-accessibility-gaps).
- **`role="group"` is set automatically on the host** and is not meant to be overridden. Remove any `role` attribute you previously added. See [step 5](#5-fix-consumer-facing-accessibility-gaps).
- **Drag and drop state is now announced to assistive technology.** A visually hidden status region announces "File ready to drop", "File accepted", and "Drop to replace existing file"; no consumer action is needed, but verify the announcements fire correctly with a screen reader in your app's own DOM structure.
- **Keep a control reachable once `filled` is set**, such as a "Replace file" button, since the default slot's browse button is hidden while `filled` is `true`. See [step 4](#4-move-uploaded-state-content-into-the-filled-content-slot).
- **`aria-dropeffect` and `aria-grabbed` must not be used.** Both are deprecated as of ARIA 1.1.
