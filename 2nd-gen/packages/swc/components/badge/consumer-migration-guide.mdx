# Badge consumer migration guide

Replace `<sp-badge>` with `<swc-badge>` and update the import. The public API is unchanged.

## What changed

| Area       | Change                                                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| Tag        | `sp-badge` â†’ `swc-badge`                                                                                                                       |
| Import     | `@spectrum-web-components/badge` â†’ `@adobe/spectrum-wc/badge`                                                                                  |
| Attributes | `variant`, `size`, `fixed` unchanged                                                                                                           |
| Default    | `variant` defaults to `informative`                                                                                                            |
| New in S2  | `subtle`, `outline`; colors `pink`, `turquoise`, `brown`, `cinnamon`, `silver`                                                                 |
| Constraint | `outline` applies only to semantic variants (`accent`, `informative`, `neutral`, `positive`, `notice`, `negative`); invalid combinations warn. |
| Slots      | Default and `icon` unchanged                                                                                                                   |
| Events     | None                                                                                                                                           |

## Update your code

### 1. Update the import

```js
// Before
import '@spectrum-web-components/badge/sp-badge.js';
// After
import '@adobe/spectrum-wc/badge';
```

### 2. Rename the tag

```html
<!-- Before -->
<sp-badge variant="positive">Approved</sp-badge>
<!-- After -->
<swc-badge variant="positive">Approved</swc-badge>
```

### 3. Add `aria-label` to icon-only badges

```html
<swc-badge size="m" aria-label="Approved">
  <sp-icon-checkmark-circle slot="icon"></sp-icon-checkmark-circle>
</swc-badge>
```

For icon + text badges, mark the decorative icon with `aria-hidden="true"`.

### 4. (Optional) Adopt new styles

```html
<swc-badge variant="informative" subtle>Active</swc-badge>
<swc-badge variant="positive" outline>Approved</swc-badge>
```

## Styling

2nd-gen exposes a new set of public CSS custom properties on `<swc-badge>`. The 1st-gen `--mod-badge-*` properties **do not apply** and must be replaced if your app used them.

Public custom properties:

- `--swc-badge-background-color`
- `--swc-badge-label-icon-color`
- `--swc-badge-border-color`
- `--swc-badge-corner-radius`
- `--swc-badge-font-size`
- `--swc-badge-line-height`
- `--swc-badge-height`
- `--swc-badge-gap`
- `--swc-badge-padding-block`
- `--swc-badge-padding-inline`
- `--swc-badge-with-icon-padding-inline`
- `--swc-badge-icon-size`
- `--swc-badge-outline-background-color`, `--swc-badge-outline-label-icon-color` (outline variants only)

Do not target internal classes, `--_swc-badge-*` private properties, or shadow DOM â€” none are public API.

## Accessibility

- **Icon-only badges** must have `aria-label` on the host.

  ```html
  <swc-badge size="m" aria-label="Approved">
    <sp-icon-checkmark-circle slot="icon"></sp-icon-checkmark-circle>
  </swc-badge>
  ```

- **Decorative icons** paired with text should use `aria-hidden="true"` so screen readers don't announce duplicate content.

  ```html
  <swc-badge variant="positive">
    <sp-icon-checkmark-circle
      aria-hidden="true"
      slot="icon"
    ></sp-icon-checkmark-circle>
    Approved
  </swc-badge>
  ```

- Badges are non-interactive. Refactor any focusable or clickable badges to `<swc-action-button>`, `<swc-tag>`, or a native button.
