Link presentation uses **native `<a href>` elements** with BEM classes from `@adobe/spectrum-wc/link.css`. Default anchor appearance inside prose and link lists ships with `@adobe/spectrum-wc/typography.css`; see [Typography](/docs/components-typography--docs) for [Prose container](/docs/components-typography--docs#prose-container) and [Link list](/docs/components-typography--docs#link-list) patterns.

For bare, classless `<a href>` elements outside typography wrappers, import `@adobe/spectrum-wc/global-link.css` to opt in to application-wide baseline link styling.

## Options

### Standalone

Explicit-context links use `.swc-Link` with `.swc-Link--standalone` for anchors outside prose wrappers. `.swc-Link` is the modifier used to apply the primary variant (default link styling to anchors.

<Canvas of={LinkStories.Standalone} />

### Secondary

The `.swc-Link--secondary` variant applies a secondary color treatment.

<Canvas of={LinkStories.Secondary} />

### Quiet standalone

The `.swc-Link--quiet` modifier removes the default underline until hover. It requires `.swc-Link--standalone` and is intended for section-scoped patterns such as footers, not undifferentiated body copy.

<Canvas of={LinkStories.QuietStandalone} />

### Static colors

Use `.swc-Link--staticWhite` and `.swc-Link--staticBlack` when displaying links over images or colored backgrounds:

- **Static white**: Use on dark or colored backgrounds for better contrast
- **Static black**: Use on light backgrounds for better contrast

<Canvas of={LinkStories.StaticColors} />

### Link list

Use `<ul class="swc-Typography--links">` with unclassed `<a href>` in each item. Link lists in footers, sidebars, and navigation regions receive default styling from the wrapper. See [Typography / Link list](/docs/components-typography--docs#link-list).

## Accessibility

### Features

Native links provide accessible behavior without a custom element wrapper:

#### Implicit link role

- Valid `<a href>` elements expose the **link** role in the accessibility tree
- Do not add `role="link"` on anchors or use non-anchor elements for navigation

#### Accessible name

- Visible link text provides the accessible name in most cases
- For icon-only links, provide `aria-label` or equivalent text alternatives

#### Keyboard and focus

- Links participate in the tab order and activate with Enter (platform conventions apply)
- Focus-visible styles must remain perceivable; do not remove focus treatment without a deliberate replacement

#### Quiet styling

- Use quiet (non-underline) link styles only in **sections** such as footers where the surrounding region makes navigation intent clear
- In long body prose, use the default underlined link treatment from Typography

#### No disabled state

- Navigational links cannot be disabled like form controls
- When an action is unavailable, use a disabled `<button>` with explanation or remove the link

### Best practices

- Write descriptive link text that makes sense out of context where possible
- Do not rely on color alone to identify links (WCAG 1.4.1)
- Ensure sufficient contrast between link text and surrounding content (WCAG 1.4.3)
- Use real semantic structure (`p`, `ul`, `nav`, headings) around links
- Import `@adobe/spectrum-wc/typography.css` for prose and link-list defaults; add `@adobe/spectrum-wc/link.css` when explicit modifiers are needed; add `@adobe/spectrum-wc/global-link.css` for application-wide styling of bare, classless anchors

## API

Apply classes from `@adobe/spectrum-wc/link.css` on native `<a href>` elements. Default appearance inside prose and link lists does not require these classes; use [Typography](/docs/components-typography--docs) wrappers instead.

### CSS classes (`link.css`)

| Class                   | Description                                                                                                                    |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `swc-Link`              | Base link presentation for explicit-context anchors. Pair with modifiers as needed.                                            |
| `swc-Link--standalone`  | Standalone link typography (medium weight). Use on anchors outside prose wrappers.                                             |
| `swc-Link--secondary`   | Secondary (neutral) color treatment.                                                                                           |
| `swc-Link--quiet`       | Removes the default underline until hover. Must be used with `swc-Link--standalone` (section-scoped patterns such as footers). |
| `swc-Link--staticWhite` | Static white link color for dark or colored backgrounds.                                                                       |
| `swc-Link--staticBlack` | Static black link color for light backgrounds.                                                                                 |

### Typography wrappers (default link styling)

Import `@adobe/spectrum-wc/typography.css`. Child `<a href>` elements receive default link styles without `swc-Link` classes.

| Class                   | Description                                                                        |
| ----------------------- | ---------------------------------------------------------------------------------- |
| `swc-Typography--prose` | Default link appearance inside running text and semantic prose elements.           |
| `swc-Typography--links` | Default link appearance for anchors in link lists (footers, sidebars, navigation). |

### CSS custom properties

Modifiers set these `--swc-link-*` properties on the anchor. Override them for custom theming.

| Property                           | Description                      |
| ---------------------------------- | -------------------------------- |
| `--swc-link-text-color-default`    | Default link text color          |
| `--swc-link-text-color-hover`      | Link text color on hover         |
| `--swc-link-text-color-down`       | Link text color when active      |
| `--swc-link-text-color-focus`      | Link text color on focus-visible |
| `--swc-link-focus-indicator-color` | Focus ring color                 |
