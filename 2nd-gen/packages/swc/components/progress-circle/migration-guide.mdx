# Progress circle migration guide

Replace `<sp-progress-circle>` with `<swc-progress-circle>`, update the import, and use `progress` omission instead of the `indeterminate` attribute.

## Installation

```bash
# Remove
yarn remove @spectrum-web-components/progress-circle

# Add
yarn add @adobe/spectrum-wc
```

Update your import:

```js
// Before
import '@spectrum-web-components/progress-circle/sp-progress-circle.js';

// After
import '@adobe/spectrum-wc/components/progress-circle/swc-progress-circle.js';
```

> `@adobe/spectrum-wc` is a monolithic package. Importing via subpath (e.g. `@adobe/spectrum-wc/components/progress-circle/swc-progress-circle.js`) registers and loads only that component's bundle.

## What changed

### Renamed

| Area             | Spectrum 1 (`sp-progress-circle`)          | Spectrum 2 (`swc-progress-circle`)                                     |
| ---------------- | ------------------------------------------ | ---------------------------------------------------------------------- |
| Tag              | `sp-progress-circle`                       | `swc-progress-circle`                                                  |
| Import path      | `@spectrum-web-components/progress-circle` | `@adobe/spectrum-wc/components/progress-circle/swc-progress-circle.js` |
| CSS custom props | `--mod-progress-circle-*`                  | `--swc-progress-circle-*` (see [Styling](#styling))                    |

### Added in Spectrum 2

| Addition               | Notes                                                                      |
| ---------------------- | -------------------------------------------------------------------------- |
| `static-color="black"` | Use on light static backgrounds; `white` was the only option in Spectrum 1 |

### Removed in Spectrum 2

| Removed                                     | Replacement                                                                                  |
| ------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `indeterminate` boolean attribute           | Omit the `progress` attribute (or set it to `null`) â€” see [step 2](#2-replace-indeterminate) |
| Light DOM children rendered as label        | Use the `label` attribute â€” see [step 3](#3-move-label-text-to-the-label-attribute)          |
| `--mod-progress-circle-*` custom properties | `--swc-progress-circle-*` equivalents (see [Styling](#styling))                              |

## Update your code

### 1. Rename the tag

```html
<!-- Before -->
<sp-progress-circle progress="50" label="Uploading"></sp-progress-circle>
<!-- After -->
<swc-progress-circle progress="50" label="Uploading"></swc-progress-circle>
```

### 2. Replace `indeterminate`

The `indeterminate` boolean attribute has been removed. Omitting the `progress` attribute is now equivalent â€” the component defaults to indeterminate mode when no `progress` value is set. You can also set `progress` to `null` explicitly to return to indeterminate mode after a value has been assigned.

```html
<!-- Before -->
<sp-progress-circle indeterminate label="Loading"></sp-progress-circle>

<!-- After: omit progress entirely -->
<swc-progress-circle label="Loading"></swc-progress-circle>

<!-- After: or set progress to null explicitly (useful when toggling state in JS) -->
<swc-progress-circle label="Loading"></swc-progress-circle>
```

```js
// Return to indeterminate after a value has been set
progressCircle.progress = null;
```

If you set `indeterminate` and `progress` simultaneously in Spectrum 1, remove `indeterminate` and keep `progress` to render a determinate indicator.

### 3. Move label text to the `label` attribute

In Spectrum 1, text placed as a light DOM child of `<sp-progress-circle>` was rendered as a visible label. In Spectrum 2, light DOM children are not rendered. Use the `label` attribute (or `aria-label`) to provide the accessible name.

```html
<!-- Before -->
<sp-progress-circle progress="75">Saving document</sp-progress-circle>

<!-- After -->
<swc-progress-circle
  progress="75"
  label="Saving document"
></swc-progress-circle>
```

### 4. (Optional) Use `static-color="black"` for light backgrounds

The `static-color` attribute now accepts `"black"` in addition to `"white"`. Use it when placing the component on a light static background that falls outside the normal theme.

```html
<!-- On a dark background -->
<swc-progress-circle static-color="white" label="Loading"></swc-progress-circle>

<!-- On a light static background (new in Spectrum 2) -->
<swc-progress-circle static-color="black" label="Loading"></swc-progress-circle>
```

## Accessibility

- Always provide a `label` attribute (or `aria-label`) with a specific description of what is loading. The component applies a fallback `aria-label="Loading"` when no accessible name is found, but a task-specific label ("Uploading document", "Processing payment") is strongly preferred.
- `<swc-progress-circle>` sets `role="progressbar"` automatically. Do not add a conflicting `role` attribute.
- In determinate mode, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, and a locale-aware `aria-valuetext` are set automatically. Do not set these manually.
- In indeterminate mode, all `aria-value*` attributes are removed so assistive technology correctly treats the component as having no known progress value.
- Do not wrap `<swc-progress-circle>` in an `aria-live="assertive"` region â€” it will interrupt screen reader users continuously. Use `aria-live="polite"` only for deliberate status updates.

### Removed `--mod-progress-circle-*` properties

Replace every `--mod-progress-circle-*` property with its `--swc-progress-circle-*` equivalent:

| Removed (Spectrum 1)                                       | Replacement (Spectrum 2)                                                                        |
| ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `--mod-progress-circle-size`                               | `--swc-progress-circle-size`                                                                    |
| `--mod-progress-circle-thickness`                          | `--swc-progress-circle-thickness`                                                               |
| `--mod-progress-circle-track-border-color`                 | `--swc-progress-circle-track-border-color`                                                      |
| `--mod-progress-circle-fill-border-color`                  | `--swc-progress-circle-fill-border-color`                                                       |
| `--mod-progress-circle-track-border-color-over-background` | No replacement â€” use the `static-color` attribute instead (`static-color="white"` or `"black"`) |
| `--mod-progress-circle-fill-border-color-over-background`  | No replacement â€” use the `static-color` attribute instead (`static-color="white"` or `"black"`) |
| `--mod-progress-circle-track-border-style`                 | No replacement â€” S2 uses SVG strokes; border style is not configurable                          |
| `--mod-progress-circle-position`                           | No replacement                                                                                  |
