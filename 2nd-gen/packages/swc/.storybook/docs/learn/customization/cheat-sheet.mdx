# Customization cheat sheet

A scannable quick reference for customizing 2nd-gen Spectrum Web Components. For depth, follow the links into the long-form docs that sit below this page in the sidebar — this cheat sheet is the index, not a replacement.

## At a glance

2nd-gen SWC customization is **CSS, not JavaScript context**. There is no `<sp-theme>` component to wrap your app in. Instead, you:

1. Include the SWC stylesheet once at the application level
2. Set theme and scale via **classes** (`.swc-theme--dark`, `.swc-theme--sizeL`) on whatever element should govern that scope
3. Override visual details via **CSS custom properties** named `--swc-*` (per-component or globally)

That's the whole model. Everything else is detail.

## Common one-liners

### Include the stylesheet

```html
<link rel="stylesheet" href="node_modules/@adobe/spectrum-wc/dist/swc.css" />
```

Or via a JS import if your bundler supports CSS imports:

```js
import '@adobe/spectrum-wc/swc.css';
```

### Set the app background

```css
:root {
  --swc-application-background: var(--swc-gray-100);
}
```

Use a Spectrum color token so the background adapts to theme automatically.

### Switch to dark theme

```html
<body class="swc-theme swc-theme--dark">
  <!-- your app -->
</body>
```

Available theme classes: `swc-theme--light`, `swc-theme--dark`, `swc-theme--adaptive` (follows OS setting).

### Switch to large scale

```html
<body class="swc-theme swc-theme--sizeL">
  <!-- your app — components inside render at the L scale -->
</body>
```

Available scale classes: `swc-theme--sizeM` (default), `swc-theme--sizeL`. Scope is per-element — nest different scales in different parts of your app if you need to.

### Override a single component custom property

```css
/* Increase font size on extra-large badges only */
swc-badge[size='xl'] {
  --swc-badge-font-size: var(--swc-font-size-400);
}
```

Always scope to the variant/state you're targeting (`[size='xl']`, `[variant='positive']`, etc.). Bare element selectors are too broad and cause unintended overrides.

### Override a design token globally

```css
:root {
  --swc-accent-color-1000: var(--swc-blue-1100);
}
```

Replaces the accent color across all components. Use sparingly — global overrides apply everywhere.
