# What's new in Gen2?

SWC Gen2 (also called "2nd gen" or "next gen") is the newest version of Spectrum Web Components. It includes full-fidelity Spectrum 2 styles, improved system architecture, and updates to components' APIs and accessibility contracts. This page is an overview of how the packages differ so teams know what to expect when adopting or migrating.

## Key differences

| Area               | SWC Gen1                                                                                     | SWC Gen2                                                                          |
| ------------------ | -------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| Package name       | `@spectrum-web-components`                                                                   | `@adobe/spectrum-wc`                                                              |
| Documentation site | [opensource.adobe.com/spectrum-web-components](opensource.adobe.com/spectrum-web-components) | [spectrum-web-components.adobe.com](spectrum-web-components.adobe.com)            |
| Design             | Includes Spectrum, Express, and S2 Foundations themes                                        | Spectrum 2 only                                                                   |
| Architecture       | Earlier component patterns; less consistent                                                  | More consistent structure and APIs across the library                             |
| Styling            | Components are styled via `<sp-theme>`                                                       | Components styled via CSS classes at application root                             |
| Customization      | Done via `--mod` CSS custom properties                                                       | Done via `--swc` custom properties and/or global element styles                   |
| Accessibility      | Basic support, several known bugs                                                            | Improved accessibility coverage, bugs fixed                                       |
| Performance        | More interdependencies, larger bundles                                                       | Smaller bundles, higher reliance on browser standards, better runtime performance |
| Long-term support  | No new components or features, minimal bug fixes                                             | New features, components, and fixes                                               |

## What this means for teams

  <sp-accordion allow-multiple size="m" class="docs-accordion">
    <sp-accordion-item label="If you are starting a new project">
      Use Gen2 where available. It matches the current Spectrum guidelines and
      offers better performance and consistency.
    </sp-accordion-item>
    <sp-accordion-item label="If your project currently uses Gen1 SWC">
      You should begin to plan how you'll adopt gen2 components as they become
      available. Each component has a migration guide which includes more
      details on the specific changes. Migrating on a per-component basis allows
      teams to update gradually, and the gen1 S2 Foundations theme allows for
      visual consistency between both generations.
    </sp-accordion-item>
  </sp-accordion>
</sp-theme>
